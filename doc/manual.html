1 Einleitung



Das Benutzerhandbuch von ILIAS3 gliedert sich in drei Abschnitte, entsprechend der Benutzerzielgruppen:

  • Die Benutzerdokumentation erklärt den Umgang mit ILIAS3 für die verschiedenen Benutzertypen wie Administrator, Autor, Lerner usw.

  • Die Administratordokumentation zeigt Voraussetzungen und die Installation von ILIAS3. Die Managementkonsole wird in diesem Abschnitt genauer beschrieben.

  • Die Entwicklerdokumentation informiert den Programmierer über Konzepte, Datenstrukturen und Programmierung von ILIAS3. Bitte lesen Sie in diesem Teil über Hintergrundinformationen von ILIAS3.

3 Administratordokumentation



3.1 Voraussetzungen



ILIAS3 benötigt einige Voraussetzungen auf dem Server-Rechner, um ordnungsgemäß zu laufen. Sie benötigen einen Webserver (empfohlen: Apache), eine aktuelle PHP-Version (ab Version 4.2.3) und die aktuellste PEAR-Version (PHP-Framework, Funktions-Bibliothek), eine SQL-Datenbank (empfohlen MySQL). Weiterhin sollten die Mailfunktionen von PHP aus verfügbar sein. An dieser Stelle betrachten wir die Installation der PEAR-Komponenten und die Installation von ILIAS3 selbst.

Auf die anderen Komponenten (Apache, z.B. mySQL, usw) soll an dieser Stelle nicht weiter eingegangen werden. Bitte schauen Sie in die entsprechenden Dokumentationen der einzelnen Komponenten.

3.2 Installation




3.2.1 Installation von PEAR



Die Dokumentation von PEAR finden Sie unter http://pear.php.net. Die Module selbst bekommen Sie aus dem CVS. Bitte loggen Sie sich dazu in das Pear-cvs ein:

$ cvs -d :pserver:cvsread@cvs.php.net:/repository login

Das Passwort ist phpfi.

Jetzt können Sie das DB-Modul mithilfe von

$ cvs -d :pserver:cvsread@cvs.php.net:/repository co pear/DB

downloaden. Bitte kopieren Sie den Inhalt des DB-Verzeichnisses in Ihr pear-Verzeichnis auf dem Webserver, so dass Sie die DB.php und das DB-Verzeichnis überschreiben.

Das Auth Modul bekommen sie mithilfe von

$ cvs -d :pserver:cvsread@cvs.php.net:/repository co pear/Auth

Bitte kopieren Sie auch hier das Auth-Verzeichnis so, dass der Auth-Ordner direkt im pear-Verzeichnis liegt.

Bitte verwechseln Sie nicht das pear-Verzeichnis mit dem PEAR-Verzeichnis. Das PEAR-Verzeichnis ist ein Unterordner ihrer PEAR-Bibliothek im pear-Verzeichnis.

3.2.2 Installation von ILIAS3


Éntpacken Sie das mitgelieferte Archiv in das gewünschte Web-Verzeichnis unter ihrem Webroot (z.B. /var/www/ilias3/, im Folgenden abgekürzt mit …/ilias3/).
Rufen Sie unter ihrem Internetbrowser die Datei setup.php auf.

Folgen Sie den Anweisungen auf dem Bildschirm für eine automatische Installation von ILIAS3.

3.2.2.1 Manuelles Setup


Für den Fall, dass es nicht möglich ist, ILIAS3 automatisch zu installieren (INI-Datei schreiben nicht möglich, Datenbankzugangsdaten nicht griffbereit) können Sie auch ein manuelles Setup durchführen.

Schreiben der Ini-Datei
Kopieren Sie die Datei ilias.master.inc.php nach ilias.inc.php. Passen Sie dort die Daten, insbesondere Datenbankzugangsdaten, an Ihr System an.

Einrichtung der Datenbank
Erstellen Sie eine Datenbank mit dem Namen, den Sie auch in der INI-Datei eingetragen haben. Spielen Sie in diese Datenbank das SQL-Datenbank-Template im sql-Unterverzeichnis Ihrer ILIAS3-Installation ein.

Sprachdateien
Das Sprachkonzept unterliegt einem Mechanismus, der Schreibrechte auf das Sprachenverzeichnis …/ilias3/lang benötigt. Bitte ermöglichen Sie Schreibzugriff für den Webserver.

3.2.2.2 Abschluss der Installation


Falls Ihre Installation erfolgreich war, zeigt Ihnen das ILIAS3-Setup durch folgenden Bildschirm an:

Sie können sich nun auf der Startseite von ILIAS3 (also z.B. http://localhost/ilias3/) unter folgendem Benutzernamen anmelden:
Login: admin, Passwort: admin.
Nun befinden Sie sich auf der Hauptseite von ILIAS3, dem Persönlichen Schreibtisch. Bitte löschen Sie nun die Datei setup.php oder verhindern sie durch

$ chmod 000 setup.php

den Zugriff Dritter auf die Setup-Datei.
Sie haben jetzt die Möglichkeit, ILIAS3 zu administrieren, Benutzer anzulegen, etc.

4 Entwicklerdokumentation



Aufgabe dieses Kapitels ist es, dem Programmentwickler das Gesamtkonzept von ILIAS3 zu erläutern. Die Klassenstrukturen sind vereinheitlicht. Bitte halten Sie sich insbesondere an die Techniken im Konstruktor der jeweiligen Klasse (s. Abschnitt 4.2.1).
ILIAS3 basiert auf der Programmiersprache PHP und dem „neuen“ PEAR-Framework. Das Programm benutzt dabei folgende Klassen von PEAR:

  • Datenbankabstraktionsschicht (DB.php)

  • Authentifizierung (Auth.php)

  • Templateklasse IntegratedTemplate/Extension (HTML/ITX.php)



ILIAS3 umfasst neben dem Hauptzweck, der Lernumgebung, folgende Module:

  • Mail-System für interne Mail (Nachrichtensystem, das bei Bedarf auch echte Mail verschickt)

  • Sprachenkonzept

  • Forensystem zum Informationsaustausch unter den Benutzern

  • RBAC (Rollenbasierte Zugriffskontrolle)

  • Administrationskonsole zur Verwaltung des Rechtesystems

  • Logger zur Fehlerbehandlung bzw. Protokollfunktionen

  • Datenbankwrapper zur Kapselung von unterschiedlichen Datenbanken

  • Datenbankupdater zur Konsistenzerhaltung der DB während der Entwicklung

  • Fehlermanagement

  • Templatesystem zur individuellen Anpassung von ILIAS

  • Kapselung der Grundeinstellungen in eine INI-Klasse




4.1 Konzeptbeschreibung



Die folgenden Unterkapitel erklären Teilkonzepte von ILIAS3. Zum nachvollziehen der Konzepte schauen Sie bitte in die Quelltexte der jeweiligen Klassen.


4.1.1 Setup-Routine


ILIAS3 wird über eine Setup-Routine auf dem Computer installiert. Dabei können folgende Aufgaben automatisiert abgearbeitet werden:

  • Schreiben der INI-Datei (s. auch Abschnitt 4.1.2)

  • Einrichten der Datenbank auf dem Datenbankserver

  • Installation der Sprachdateien



ILIAS3 zeitg eine erfolgreiche Installation an und bietet in dem Fall den Link zum Login an.
Damit die Zahl der Fehler minimiert wird, werden im Vorfeld Voraussetzungen überprüft:

  • schreibbares ILIAS-Verzeichnis (wird benötigt für die ini-Datei),

  • schreibbares Sprachenverzeichnis für die Sprachdateien.


Sind diese Voraussetzungen nicht gegeben, bietet ILIAS3 keine automatisierte Installation.
Weitere Installationsschritte können dabei bequem über Anpassung des Setup-Templates, der Setup-Klasse und des Setup-Skriptes vorgenommen werden.
Neben den Überprüfungen auf schreibbare Verzeichnisse etc. besteht die Hauptaufgabe der Setup-Klasse in der korrekten Einrichtung der Datenbank. Dazu ist die neueste PEAR-Version nötig. Die Setup-Klasse fängt beim Connect zum Datenbankserver Fehler ab und berichtet diese.

4.1.2 INI-Datei-Handling


Die IniFile-Klasse kapselt Lese- und Schreibzugriff auf die Ini-Datei von ILIAS3 (ilias.ini.php). Durch sie werden Fehler beim Lesen abgefangen und berichtet. Die wichtigste Methode für den Programmierer ist readVariable. Eine Ini-Instanz befindet sich im Ilias-Hauptobjekt $ilias-ini. Sie können z.B. die Default-Systemsprache über folgenden Aufruf auslesen:
$lng = $ilias->ini->readVariable(„language“, „default“);
Dabei bezeichnet der erste Parameter die Gruppe, der zweite Parameter der Variablenname. Gruppen erkennen Sie in der Ini-Datei an den eckigen Klammern (z.B. [language]).

4.1.3 Fehlermanagement


ILIAS3 bietet ein Fehlermanagement für die Behandlung von fatalen Fehlern (sofortiger Programmabbruch), Warnungen (Fehler mit sofortigem Programmabbruch und Ausgabe auf einer Fehlerseite), und Message (Ausgabe einer Nachricht auf dem Bildschirm bei fortlaufendem Programm). Dazu wird in allen Klassen das Fehlermanagement von ILIAS3 aktiviert, dass durch die Klasse ErrorHandling repräsentiert wird. Diese Klasse wird im Ilias-Objekt instanziert, ruft den PEAR-Errorhandler auf und behandelt die Fehler je nach aufgetretenem Fehlergrad.

4.1.3.1 Erzeugen von Fehlern


Falls in einer Klasse ein Fehler aufgetreten ist, wird er entweder automatisch behandelt (im Falle von Datenbankabfragen - ungültige SQL-Statements) oder muss von Ihnen explizit erhoben werden. Dafür steht ihnen die Methode $this->raiseError zur Verfügung. Geben Sie als ersten Parameter die Nachricht des Fehlers an, als zweiten Parameter erwartet die Methode die Art des Fehlers, z.B.
$this->raiseError(„user exists“, $this->FATAL);

4.1.4 Datenbankwrapper


ILIAS3 leitet die PEAR-Klasse DB noch einmal ab, damit der Fehlermechanismus automatisch ablaufen kann. Schlägt eine Abfrage ($this->query) fehl, so wird automatisch ein Fataler Fehler erzeugt und eine entsprechende Meldung auf dem Bildschirm ausgegeben (error.php).

4.1.5 Datenbankupdater


ILIAS3 ist mit einem Mechanismus ausgestattet, der die Datenbank während des Entwicklungsprozesses eines Releases unter allen Entwicklern up-to-date hält. Ändert man etwas an der Datenbank, so werden die Änderungen in dbupdate.php im sql-Verzeichnis eingetragen. Bei der Pflege des Entwicklungssnapshots (also der aktuellen Developerversion) kann man nun im Adminbereich diese Datanbankversion überprüfen und evtl. updaten. Vor Veröffentlichung eines neuen Releases wird ILIAS3 auf einem Testsystem installiert, vorhandene Updates installiert und daraufhin die Datenbank exportiert. Dieser Datenbankauszug (dump) ersetzt das alte Datenbanktemplate (ilias3.sql).
Folgende Regeln sind zu beachten, wenn Datenbankänderungen in das Skript dbupdate.php eingetragen werden:

  • Inkrementieren Sie die Versionsnummer der letzten Änderung und fügen Sie diese darunter ein. (Format: <#nummer>).

  • SQL-Statements müssen mit einem Semikolon abgeschlossen werden. Inhalte von Textfeldern sind in einfache Anführungszeichen zu setzen.

  • Sie können auch PHP-Code verwenden, der mit <?php ?> umschlossen sein muss.


Das Ilias-Objekt stellt Ihnen einen Datenbank-Handler zur Verfügung ($ilias->db).

4.1.6 Loggingmechanismus


ILIAS3 bietet einen einfachen Loggingmechanismus in eine Datei. Sie haben Zugriff auf den Logger mittels des Ilias-Objektes:
$ilias->log->write($msg);
Daraufhin wird in die Datei ilias.log die Nachricht $msg geschrieben. Der Logger eignet sich besonders gut zum Debugging. Z.B. wird er eingesetzt, um den Administrator über fehlende Sprachtopics zu informieren.

4.1.7 Interne Mail


Ein internes Mailsystem gehört zum Lieferumfang von ILIAS3. Dieses Mailsystem besteht aus einer Klasse.

4.1.8 Sprachkonzept


ILIAS3 ist sprachenunabhängig, d.h. sämtliche Texte sind in Templates mittels definierten Platzhaltern angegeben (z.b. TXT_SAVE) die in den aufgerufenen PHP-Skripten in die jeweilige Benutzersprache übersetzt werden. Eine Übersetzung eines solchen Platzhalters geschieht mit z.B. mit folgendem Befehl:
$tpl->setVariable(„TXT_SAVE“, $lng->txt(„save“));
Der Zugriff auf die Benutzersprache wird in der header-Datei von ILIAS3 initialisiert. Die Sprachklasse (class.Language.php) wird dabei instanziert.
$lng = new Language($ilias->account->data[„language“]);
$data[language] besteht meist aus einem zweibuchstabigen Sprachcode („en“, „de“, „it“, …). Die Sprachklasse versucht nun, auf eine generierte Sprachdatei zuzugreifen, die im lang-Ordner von ILIAS3 liegt. Schlägt dies fehl, wird keine Sprache initialisiert.


4.2 Programmierung


Die Programmierung von ILIAS3 unterliegt einigen Regeln. Näheres hierzu erfahren Sie unter rules.txt sowie unter Programmierrichtlinien.doc. Dort sind Variablennamenkonventionen u.v.m. aufgeführt. Auf folgendes möchten wir kurz eingehen:
Formularbehandlung
Es gibt in Formularen feste Parameter, wie $id, die vom Benutzer nicht verändert werden können. Anstelle diese in hidden-Felder abzulegen, werden diese per GET-Methode in der Action des Formulars eingetragen, Beispiel:
<form name=“form“ action=“{FORMACTION}“ method=”post”>
bzw.
$tpl->setVariable(“FORMACTION”, “admusers.php?id=“.$id);
Die Variablen werden anschließend in den Arrays $_POST und $_GET behandelt. So ist weiterhin ein gezieltes Ansteuern von anderen Seiten ohne Submit möglich. Der Code ist übersichtlicher, da nach User-Submitted Variablen und festen Variablen unterschieden wird.
So ist es möglich, „Register Globals = off“ zu setzen.

5 Templatedokumentation


Ein großer Fortschritt von ILIAS3 gegenüber Vorgängerversionen besteht in der Verwendung von Templates.
Personen mit Shell-/FTP-Zugang zum Rechner können neue Templatesets definieren, bzw. vorhandene Templatesets umdefinieren. Dadurch kann der Benutzer ILIAS an seine Bedürfnisse anpassen, z.B. ILIAS auch über einen PDA benutzen.
Sofern vorhanden kann der Benutzer in einem Templateset zwischen verschiedenen Farbensets wählen, die durch Stylesheet-Dateien (*.css) definiert wurden. Ist kein Set gewählt, wird standartmäßig automatisch das erste Set übernommen.

5.1 Stylesheet-Dateien


Im Template-Ordner "default" befinden sich nach der Installation zwei Stylesheet-Dateien: "style.css" und "bluewonder.css". Diese können im persönlichen Profil ausgewählt werden. Dazu muss das Template "usr_profile.html" die Selectbox "usr_style" zur Auswahl der Styles enthalten.
Sie können beliebig viele Stylesheets hinzufügen, beachten Sie dabei bitte die Korrektheit der Dateien. Die Dateien sollten alle eigendefinierten Elemente (die mit ".") enthalten, wie in den Musterdateien angegeben.
Eine Liste der Tags:

  • body, td, h1, h2, a, a:hover, a:visited

    • standard-Formatierung von normalen HTML-Elementen
  • .btnbar Die Titelzeile von Ilias mit den Hauptmenüelementen

  • .tblheader Titelzeile einer Tabelle

  • .tblrow1, .tblrow2 Tabellenzeilenformatierung für wechselnde Farben

  • a.mailread Formatierung für gelesene Mail

  • a.mailunread Formatierung für ungelesene Mail

  • .btn Ein Knopf in ILIAS

  • a.btn Link für die Beschriftung des Knopfes

  • a.btn:hover Link für Mouseovereffekt



Sie können selbstverständlich weitere Formatierungen hinzufügen und die vorhandenen beliebig ändern.
Durch eine Vielzahl von Stylesheet-Dateien erreichen sie eine höhere Benutzerakzeptanz.


5.2 Framesets und nicht-Framegestützte Seiten


ILIAS3 ist nicht auf Frames festgelegt. Das "default"-Templateset ist ein Vorlagensatz mit Frames, "noframes" bezeichnet dem Namen nach ein Templateset ohne Frames (mit Hauptmenü an der Seite).
ILIAS3 erkennt selbständig framegestützte Seiten an der Datei "tpl.start.html". Diese Datei definiert das Hauptframeset von ILIAS und verweist auf das Hauptmenü und auf den Persönlichen Schreibtisch (usr_personaldesktop.php). Betrachten Sie die Datei tpl.start.html für weitere Details.
Ein weiteres Frameset kann bei Aufruf der Objekt-Administration bzw. bei Aufruf der Lerneinheiten erfolgen.
Dieses Frameset wird durch die Dateien "tpl.lo.html" bzw. "tpl.adm.html" definiert. Für weitere Einzelheiten öffnen Sie bitte diese Dateien.

5.3 Template-Dateien


ILIAS§ benötigt eine Reihe von html-Dateien, die das Layout von IlLIAS3 bestimmen. Diese Dateien finden Sie im jeweiligen Templateordner. In den Dateien gibt es eine Anzahl von "Topics". Es existieren

  • Variablen-Topics (z.B. {MAIL_COUNT}), die durch das Programm dynamisch ersetzt werden (hier: z.B. "3" um anzuzeigen, das es drei neue Mails gibt), und

  • Blöcke, die von ILIAS wiederholt werden können (z.B. eine Zeile in einer Mailübersicht) oder nur in bestimmten Fällen ausgegeben werden (z.B. eine Fehlermeldung)


Bitte halten Sie sich an die Schreibweise der Variablen bzw. Blöcke. Diese wurden in ILIAS§ definiert und können nicht verändert werden.
Alle Textanteile in den Templates werden ebenfalls durch Variablen bezeichnet, um die Vielsprachigkeit von IIAS3 zu gewährleisten. Alle Textvariablen werden durch "TXT_" vor der Variable markiert (z.B. TXT_MAIL wird zu "mail" im Englischen, zu "Posta" im Italienischen). Dieses System ist eng mit dem Sprachensystem von ILIAS3 verknüpft. Lesen Sie, falls Sie neue Sprachen hinzufügen wollen, im Abschnitt "Sprachen" (???).
Nachfolgend finden Sie eine Liste von Blöcken und Variablen in den einzelnen Templatedateien von Ilias. Global definierte Variablen (also Variablen, die in keinem Block stehen) stehen unter dem Abschnitt "global". Beachten Sie bitte, dass Blöcke aus kleinen Buchstaben bestehen, Variablenbezeichner bestehen aus Großbuchstaben.
